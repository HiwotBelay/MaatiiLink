# Contributing — SABA CODERS

## Workflow

1. Pull latest `develop`
2. Branch: `feature/<module>-<description>`
3. Open PR into `develop` — **1 teammate must review**
4. Merge only if CI passes (added in Phase 2)

## Definition of Done

- [ ] Code reviewed
- [ ] Types and lint pass
- [ ] New write operations have audit log coverage (when applicable)
- [ ] Tested locally with Docker DB running
- [ ] Docs updated if behavior changed

## Communication

- **Monday:** sprint planning (1h)
- **Wednesday:** tech sync (30m)
- **Friday:** demo to bank mentor (1h)
